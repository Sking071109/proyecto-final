# Proyecto Final — Fase 1: JavaScript
### LevelUp Code Bootcamp 2026

---

## 🎯 La misión

Construye una app web original que consuma al menos una API 
pública real. El tema es tuyo. La idea es tuya. Lo que importa 
es que funcione, que se vea bien y que puedas defenderla.

---

## 📋 Criterios de evaluación

### Funcionalidad y lógica

- Consume al menos una API pública real con async/await
- Maneja los tres estados visibles en la UI: cargando, 
  éxito y error
- Verifica response.ok y lanza errores con throw new Error()
- Procesa los datos recibidos antes de mostrarlos — al menos
  un array method del arsenal aplicado sobre la respuesta
  de la API (map, filter, find, sort, reduce o combinados)

### Buenas prácticas del curso

- Código escrito con las prácticas modernas de ES6 vistas
  en clase: arrow functions, template literals, destructuring
  donde aplique
- Si la app maneja colecciones de datos propios del usuario,
  usa localStorage para persistirlos entre sesiones
- Si la app tiene entidades con comportamiento propio
  (un carrito, un perfil, un inventario, un personaje),
  usa una clase con constructor y métodos — POO donde tenga
  sentido, no POO forzada donde no la necesitas
- Variables con nombres descriptivos, comentarios que
  expliquen el por qué no el qué

### Diseño y experiencia

- La app tiene nombre y propósito claro
- Se ve como un producto terminado, no como una práctica
- El diseño es coherente: tipografía, colores y espaciado
  consistentes en toda la interfaz
- Es responsiva — se ve bien en laptop y en móvil
- Los estados de carga y error son visibles y tienen sentido
  para el usuario, no solo para el desarrollador

### Presentación

- En la ceremonia tienes 5 minutos para mostrar la app
  funcionando y explicar qué hace y por qué la elegiste
- Debes poder responder: ¿qué API usaste?, ¿qué problema
  resuelve la app?, ¿qué fue lo más difícil de construir?

## 🔥 ¿Quieres ir más lejos?

Estas ideas no son obligatorias pero marcan la diferencia
entre una entrega buena y una entrega memorable:

- Múltiples endpoints o APIs combinadas
- Búsqueda con historial guardado en localStorage
- Filtros o sorting aplicados sobre los datos recibidos
- Modo oscuro / claro con preferencia guardada
- Animaciones y transiciones que refuercen los estados

---

## 💡 Un último consejo

Elige un tema que genuinamente te importe.
Las mejores entregas siempre vienen de alguien
que quería construir exactamente eso.
